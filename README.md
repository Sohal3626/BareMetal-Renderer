## 프로젝트 개요
요즘은 다양한 학습 자료를 쉽게 찾을 수 있는 환경이 잘 갖춰져 있습니다.  
과거에는 혼자 공부하거나 언어 장벽 때문에 어려움을 겪는 경우가 많았지만,  
지금은 AI의 도움을 받아 학습 로드맵을 구성하고 개념을 이해하는 것도 훨씬 수월해졌습니다.

이 프로젝트는 이러한 환경을 활용하여  
렌더링의 기본적인 파이프라인을 학습하기 위해  
AI의 도움과 Tiny Renderer 스타일 튜토리얼을 기반으로 진행되었습니다.

## 현재 진행 상황

### 완료된 기능
- 캔버스 저장
- Bresenham 알고리즘 기반 선 그리기
- 삼각형 래스터라이징
- 모델 파싱
- 모델 렌더링
- 플랫 셰이딩
- 단일 텍스처 매핑
- MTL 파싱 기반 다중 텍스처 매핑

### 향후 계획
- 보간 기반 고급 셰이딩 구현
- MTL 파싱을 활용한 질감 표현 개선
- 기타 렌더링 기능 확장

## Renderer 관련 설명

### Renderer 클래스
렌더링 관련 주요 기능을 제공하는 클래스입니다.  
현재는 정적 함수 형태로 구성되어 있으며,  
향후 다양한 렌더링 설정을 지원하는 구조로 확장할 가능성을 고려해 클래스 형태로 유지하고 있습니다.

---

### draw_line
Bresenham 알고리즘을 사용하여 선을 그리는 함수입니다.  
초기 구현 단계에서 작성된 기본적인 기능입니다.

---

### fill_triangle
삼각형을 래스터라이징하는 함수입니다.  

초기에는 스캔라인 방식으로 구현했지만,  
현재는 bounding box를 기준으로 barycentric 좌표를 계산하여  
삼각형 내부 픽셀을 판별하는 방식으로 변경되었습니다.

---

### draw_model
모델 데이터를 기반으로 캔버스에 렌더링을 수행합니다.  

- 변환 행렬 적용  
- 광원 기반 밝기 계산  
- 삼각형 단위 래스터라이징  

과정을 거쳐 최종 이미지를 생성합니다.

---

### transform
모델의 회전 및 카메라 위치를 행렬 연산으로 처리합니다.  

각도는 라디안 단위를 사용하며,  
행렬 연산을 통해 회전과 투영을 수행합니다.

---

### get_intensity
삼각형의 법선 벡터와 광원 벡터의 내적을 이용해  
면의 밝기를 계산합니다.

---

## 기타 개선 메모
- 일부 함수 구조 및 데이터 설계는 리팩토링이 필요합니다.
- 특히 `RenderData` 관련 구조는 책임 분리가 부족한 상태입니다.
- 향후 유지보수성과 확장성을 고려해 구조 개선이 예정되어 있습니다.
